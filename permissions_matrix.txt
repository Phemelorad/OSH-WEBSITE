================================================================================
   FORMS & PAGES — PERMISSIONS MATRIX
   Role-Based View, Submit & Edit Rights
   Roles: viewer | worker | company | medical_practitioner | officer | admin | super_admin
================================================================================


HOW TO READ THIS MATRIX
  V = Can View (page is visible in navigation / data is accessible)
  S = Can Submit (can insert new records via this form)
  E = Can Edit (can modify existing records)
  - = No access (page hidden or insert denied by RLS)

  Submit rights are enforced by BOTH nav visibility AND database RLS policies.
  Edit rights vary by record ownership (officers can edit own, admins edit all).

SECTION 7: KEY GAPS & ANOMALIES
================================================================================

  GAP-01: Medical exam insert is open to ALL authenticated users
  ────────────────────────────────────────────────────────────────────────────
  Policy: "Officers can insert medical reports" WITH CHECK (true)
  This means ANY logged-in user (viewer, worker, company) can submit a
  medical examination report. The nav hides the page from non-practitioners,
  but anyone who knows the URL can POST directly to the API.


  GAP-02: Company registration open to anyone (including unauthenticated)
  ────────────────────────────────────────────────────────────────────────────
  Policy: "Anyone can insert companies" WITH CHECK (true)
  The anon role (unauthenticated users) can insert companies!


  GAP-03: Multiple nav entries for same form with different visibility
  ────────────────────────────────────────────────────────────────────────────
  accident-report.html appears TWICE in nav.js:
    - Once under "Report Accident" (hidden from all except company)
    - Once under "Accidents > New Accident Report" (hidden from viewer, med_prac)
  This is intentional for role-based navigation but creates confusion.


  GAP-04: BL Forms have no RLS INSERT policies
  ────────────────────────────────────────────────────────────────────────────
  Only form-43-04 (incapacity) has INSERT policies defined. The other 4 BL
  form tables will DENY all insert attempts, even from admins.
  (bl_form_43_02_wages, certificate_of_insurance, medical_attendance_notifications)


  GAP-05: OHS Form 19 (Investigation) has no RLS policies
  ────────────────────────────────────────────────────────────────────────────
  The ohs_form_19 table was created without any RLS policies. Supabase
  blocks all operations by default when RLS is enabled but no policies exist.
  Officers cannot submit investigations until policies are created.
